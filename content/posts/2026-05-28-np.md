#### NP / NP-Hard / NP-Complete Problems and Related LeetCode Problems

| #    | Problem                              | Classification                  | Notes                                                        | Related LeetCode Problems                                    |
| ---- | ------------------------------------ | ------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 1    | SAT / 3-SAT                          | NP-complete                     | SAT is the first NP-complete problem. 3-SAT is the standard starting point for many NP-completeness reductions. | No direct standard LeetCode SAT problem. Related style: constraint satisfaction / backtracking problems such as LC 37 Sudoku Solver, LC 51 N-Queens |
| 2    | Clique                               | NP-complete                     | Decision version: “Does a clique of size k exist?”           | No direct standard LeetCode problem. Some graph/backtracking problems are conceptually related. |
| 3    | Vertex Cover                         | NP-complete                     | Decision version: “Is there a vertex cover of size ≤ k?”     | No direct general version. Related restricted/tree-style idea: LC 968 Binary Tree Cameras |
| 4    | Hamiltonian Cycle / Hamiltonian Path | NP-complete                     | Visit every vertex exactly once. Cycle version returns to the start. | LC 980 Unique Paths III, LC 847 Shortest Path Visiting All Nodes |
| 5    | TSP, decision version                | NP-complete                     | “Is there a tour with total cost ≤ K?”                       | LC 847 Shortest Path Visiting All Nodes, LC 943 Find the Shortest Superstring |
| 6    | TSP, optimization version            | NP-hard                         | “Find the shortest tour.” Not a yes/no decision problem, so not NP-complete. | LC 847 Shortest Path Visiting All Nodes, LC 943 Find the Shortest Superstring |
| 7    | Subset Sum                           | NP-complete                     | “Is there a subset whose sum is exactly S?”                  | LC 416 Partition Equal Subset Sum, LC 494 Target Sum, LC 698 Partition to K Equal Sum Subsets, LC 473 Matchsticks to Square |
| 8    | Partition                            | NP-complete                     | Special case of Subset Sum: split into two equal-sum subsets. | LC 416 Partition Equal Subset Sum, LC 1049 Last Stone Weight II |
| 9    | Knapsack, decision version           | NP-complete                     | “Can we reach value ≥ V within weight ≤ W?”                  | LC 416 Partition Equal Subset Sum, LC 494 Target Sum, LC 474 Ones and Zeroes, LC 879 Profitable Schemes |
| 10   | Knapsack, optimization version       | NP-hard                         | “Find the maximum value within capacity.”                    | LC 474 Ones and Zeroes, LC 879 Profitable Schemes            |
| 11   | Graph Coloring, decision version     | NP-complete                     | “Can this graph be colored with k colors?” For k ≥ 3, it is NP-complete. | LC 785 Is Graph Bipartite?, LC 886 Possible Bipartition, LC 1042 Flower Planting With No Adjacent |
| 12   | Minimum Graph Coloring               | NP-hard                         | “Find the minimum number of colors.” Optimization version.   | No direct standard LeetCode version. LC 785 and LC 886 are only 2-coloring/bipartite variants. |
| 13   | Halting Problem                      | NP-hard, undecidable, not in NP | “Given a program and input, will the program eventually halt?” No algorithm can decide this for all programs. | No LeetCode problem. This is computability theory, not coding interview material. |
