## Kubernetes in Docker



I am running language study platform in k3s, which is lightweight kubernates from Rancher with my own device(3 nodes cluster).



Someday, I wanted to run dev environment, but I don't want to run perfectly same style k3s cluster(and also I don't have more devices, only I left 2017 Mac book pro). finally I found k3d(which is developed from Rancher for testing environment)



k3d is running on docker platform, as u can see in name. Every docker containers are like nodes of k3s, not perfectly same environment, but u can take a few devices to construct full cluster.



in my case, since I am running k3d in mac book, I have to chooose docker engine well, so I chosen Colima, which is lightweight docker engine for Mac OS.



theseday, I use this k3d, testing a few things

1. Calico, which is network interface(CNI) for kubernates. this is different with default kubernates network interface(CNI) Flannel(now I am running my own service on this CNI). Calico can divide network in subnet level(default Flannel couldn't)
2. native AI-gent management(using cloud LLM services like Gemini and Codex).

I am currently running my language-learning platform on K3s, a lightweight Kubernetes distribution developed by Rancher, using my own 3-node cluster.

At some point, I wanted to create a separate development environment. However, I did not want to build another full physical K3s cluster, and I also did not have enough spare devices except for my 2017 MacBook Pro. Eventually, I discovered K3d, which is designed for lightweight testing and development environments.

As the name suggests, K3d runs K3s on top of Docker. Each Docker container behaves like a node in a K3s cluster. Although it is not a perfectly identical environment compared to a real multi-device cluster, it is still very useful for simulating Kubernetes infrastructure on a single machine.

In my case, since I run K3d on macOS, choosing the right Docker runtime was important. I decided to use Colima because it is lightweight and works well on macOS.

These days, I use this K3d environment to test several technologies:

1. Calico, which is a Kubernetes CNI (Container Network Interface) plugin.
    This is different from the default Kubernetes CNI, Flannel, which I currently use in production.
    Calico provides more advanced network controls, including subnet-level network policies that Flannel does not fully support by default.
2. Native AI-agent-based infrastructure management using cloud LLM services such as Google Gemini and OpenAI Codex.
   - So far, the results have been surprisingly good. I granted my AI agent full permissions within the K3d cluster environment.
   - I send commands through Telegram, and the AI agent creates pods, modifies cluster configurations, and manages infrastructure tasks automatically.
   - Recently, I have also been using this AI agent to generate static websites for several institutions.

I plan to continue documenting the new technologies and workflows I adopt in the future.
